HTTPS Vs. HTTP



What is the HTTP?

Before really dive into the topic of HTTPS, it is better to understand what is HTTP first.

HTTP (Hypertext Transfer Protocol) is the set of rule for transferring different files over the web, the fille including text, images, sound, and other multi-media file. It is the tool for World Wide Web (WWW) to communicate between client and servers. Therefore, it just like a language that is not used by human, but between clients (computers) and the servers.


How Does HTTP work?

All the communications are done by request and responeses. Here are the steps for a computer communicating with the servers:

  1. The client (typically a browser) send an HTTP request to the server
  2. The servers receive the request
  3. The server check whether to apply the request
  4. If approved, the server will return the HTTP response to the client
  5. The cilent receives the responses and renders the wanted content on the browser.

What's the difference between HTTP and HTTPS

We have gone through so many details about HTTP, so what really is the difference?

Most of the things are the same for both HTTP and HTTPS except encryption


Encryption

HTTPS (HTTP-Hypertext-Transfer-Protocol Secure) as its name suggested, is more secure version of HTTP. HTTPS is a much more secure and reliable protocol for transmitting sensitive information.

The uses of SSL (Secure Sockets Layer) or TLS (Transport Layer Security) protocol have been implemented for the encryption and the decryption of the data transmitted between the browser and the servers.

It is important to have these kinds of protocols for encryption when inputting sensitive data, like password. Just by using http, data can easily be captured and get all the information you input on the web. This can easily be done by some software, like Wireshark. Thus, it is necessary to have a more secure website since there are many transactions done online. No one wants to disclose their credit card number publicly.
